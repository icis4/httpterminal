Melexis.IO User Guide

Overview & Requirements

Melexis.IO is a browser-based serial terminal application for communicating with Melexis devices. It provides IR imaging, people detection, and comprehensive terminal functionality.

System Requirements

  • Browser: Chrome or Edge desktop (Web Serial API support required)
  • Connection: HTTPS or localhost (required for Web Serial access)
  • Port Access: Only one application can hold a serial port at a time
  • Permissions: Serial port access permission will be requested when connecting

Quick Start

  1. Open the app in Chrome/Edge over HTTPS or localhost
  2. Click "Connect…" button
  3. Select your device from the serial port list
  4. Grant permission when prompted
  5. Start sending commands or explore the various tabs
  6. Use "Disconnect" when finished

Tabs & Features

Terminal

  • Send/receive text commands
  • Command history with ↑/↓ navigation
  • Local echo toggle
  • Auto-scroll option
  • Clear and Save log functions
  • Quick command buttons (ID, INFO, Log, Help, Reset)
  • Reset behavior: waits for *RESET* then (OK)>, disconnects, waits 5s, and reconnects to the same device

Settings

  • Serial connection parameters (baud, data bits, parity, stop bits, flow control)
  • Input options (EOL, local echo, enter sends, auto-scroll)
  • Export/import settings as JSON
  • Reset to defaults
  • EEPROM save/load (when connected)

IR Image

  • 32×24 thermal imaging display
  • Auto-scale or manual min/max range
  • Scale options: x1, x2, x4 with bicubic interpolation
  • Grid overlay and numeric values (x1 scale)
  • Read single frame or continuous mode
  • Save PNG images and export CSV data
  • Recording functionality

People Detection

  • Parallel IR functionality with people detection overlay
  • Independent scale selector (x1/x2/x4)
  • Grid overlay and numeric values options
  • Tooltip with raw and interpolated values
  • Continuous mode with pause-on-hover
  • Separate export with "people-detection" prefix

History & Persistence

Command History

  • Stored in browser cookies for this site
  • Newest commands appear first
  • Click history buttons to resend commands
  • Navigate with ↑/↓ arrow keys in input field
  • Clear history button available in sidebar

Settings Storage

  • localStorage: Connection settings and input options
  • Export/Import: JSON bundle includes history, settings, IR parameters, and input options
  • Auto-restore: Settings automatically reload on page refresh

Export Bundle Contents

  • Command history array
  • Serial connection settings
  • IR configuration (auto-scale, min/max, scale)
  • Input options (EOL, echo, enter sends, auto-scroll)
  • Metadata (version, export timestamp)

Keyboard Shortcuts

  • Enter - Send command (when "Enter sends" is enabled)
  • ↑ - Navigate to older command in history
  • ↓ - Navigate to newer command in history
  • Tab - Navigate between UI elements
  • Space or Enter - Activate focused buttons

Troubleshooting

Common Issues

Port Not Shown

  • Ensure device is properly connected
  • Check if another application is using the port
  • Try unplugging and reconnecting the device
  • Verify device drivers are installed

Failed to Connect

  • Grant permission when browser prompts
  • Close other serial applications (Arduino IDE, etc.)
  • Check serial port settings match device requirements
  • Refresh page and try again

No Output

  • Verify correct baud rate and serial parameters
  • Check EOL (End of Line) setting matches device expectations
  • Try enabling local echo to see transmitted commands
  • Send a simple command like *IDN? to test

Browser Support

  • Web Serial API requires Chrome 89+ or Edge 89+
  • Firefox and Safari are not supported
  • Mobile browsers do not support Web Serial API
  • Ensure site is accessed via HTTPS or localhost

Permissions

  • Grant serial port access when prompted
  • Check browser permission settings if blocked
  • Clear site data and try again if permissions are corrupted

Privacy & Storage

Your privacy and data security are important. Here's what the app stores:

Local Storage

  • Command History: Stored in browser cookies, cleared with "Clear history" button
  • Settings: Connection and input preferences stored in localStorage
  • No Server Communication: All data remains on your device
  • No Telemetry: No usage data is collected or transmitted

Data Control

  • Export settings and history to JSON for backup
  • Clear history manually via UI button
  • Clear all data by clearing browser storage for this site
  • Settings reset button restores defaults
